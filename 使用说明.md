# Sync Copy · 使用说明

局域网多设备剪切板同步。一台机器上复制，所有设备都能粘贴。支持文本、图片、文件拖传。无需服务器、无需账号、无需密码。

![logo](static/app-icon.svg)

---

## 是什么 / 不是什么

**是**：
- 你在家里的 Mac 上 Cmd+C 一段代码 → 公司的 Windows 立刻能 Ctrl+V
- 手机截图 → 用 AirDrop/某工具过来？不，直接 Cmd+Shift+4 截图 → 另一台电脑按 Ctrl+V 就贴出来了
- 拖一个 PDF 到浮窗，对方同意就存到 Downloads

**不是**：
- 云剪切板（没有中心服务器，离开局域网不工作）
- 能跨互联网（只在同一 LAN / WiFi / 以太网）
- 文件仓库（单文件 5 MB 上限，不适合传大文件）

## 安装

### 下载
从仓库的 **GitHub Releases** 或 **Actions artifacts** 下载对应平台的安装包：

| 平台 | 文件 |
|---|---|
| macOS (Apple Silicon + Intel 通用) | `Sync Copy_<版本>_universal.dmg` 或 `.app` |
| Windows 10/11 x64 | `Sync Copy_<版本>_x64-setup.exe`（NSIS 便携式）或 `.msi`（Installer） |

### 安装

**macOS**：双击 `.dmg`，把 `Sync Copy.app` 拖进 Applications。如果提示「来自未识别开发者」，右键 → 打开，再确认即可。

**Windows**：双击 `-setup.exe`，跟着向导装。如果 Windows Defender / 安全卫士警告，选"仍然运行"——这是未签名桌面应用的通病，非恶意。

## 首次使用（3 步）

### 1. 在**所有**设备上启动 Sync Copy

- macOS：Launchpad / Applications 里双击
- Windows：开始菜单 → Sync Copy

启动后浮窗会自动置顶显示。状态栏（Mac 顶部 / Win 右下角）会出现一个蓝色双纸的托盘图标。

### 2. 在某一台（比如 **Mac**）上查"本机地址"

浮窗底部左下角会显示类似 `192.168.1.244:5858`。点击一下就复制了（会闪一下绿色 `已复制`）。

### 3. 在另一台（**Windows**）上「加入」

- 点浮窗顶部蓝色 `加入` 按钮
- 在输入框里贴入刚才复制的 `192.168.1.244:5858`
- 点「加入」

此时**两台设备上都会弹出一个审批框**：
```
📥 有设备希望加入
    <Windows 设备名>
    还剩 30 秒未响应视为拒绝
    [拒绝]  [同意]
```

**任意一台**点「同意」，**两台的弹框同时消失**，连接建立。状态变成绿色 `小组 · 2 台`。

## 日常使用

### 复制文本
Mac 上选一段文字 → Cmd+C → 0.5-1 秒内 Windows 浮窗里会出现这条历史 → Windows 任意地方 Ctrl+V 粘出。

### 复制图片 / 截图
- Mac：Cmd+Shift+Ctrl+4 截屏会自动放进剪切板
- Windows：Win+Shift+S → 选区截屏会放进剪切板
- 截图在对面 Word / 画图 / 微信 / 备忘录里直接 Ctrl+V / Cmd+V 粘出

### 历史记录
浮窗中间纵向列表显示最近同步过的 50 条。

- **单击条目** → 重新把这条复制到系统剪切板（短暂闪绿 + `已复制` 小标签）
- **单击文件条目** → 在 Finder / 资源管理器里打开文件位置
- **鼠标移到条目上右侧** → 出现灰色 `✕` → 单击删除该条 **并同步删除所有设备上的同一条**

### 文件拖传
把本地任何**单个 < 5 MB** 的文件拖到浮窗里松手。

- 你的浮窗闪蓝色提示「松开即发送给所有设备」
- 对方设备上弹框 `📎 收到 xxx.pdf (2.1 MB) 来自 Mac / [拒绝] [保存]`
- 对方点保存 → 文件自动存到对方系统的 **Downloads** 目录
- 同名文件自动加 `_1` / `_2` 后缀不覆盖
- 你方的历史列表里会出现一条 `📎 已发送`
- 对方的历史列表里会出现 `📎 已保存` → 点击此条可在文件管理器里定位到文件位置

### 最小化浮窗
点浮窗顶部 `−` 按钮 → 窗口缩小成一个 48×48 的圆角方形小图标（悬浮球），仍置顶。

- 拖动球 → 移动位置
- 单击球 → 展开回原窗口尺寸
- 球右下角小数字 = 当前小组在线台数
- 球右上角红色数字 = 有几个审批/文件待处理

### 完全退出
点 `⚙` 设置 → 底部红色 `退出应用` 按钮。退出前会自动通知其它设备把你摘掉，计数实时更新。

或点托盘图标右键菜单 → 退出。

## 设备加入新机器

已经有若干设备（A、B）已连接，现在 C 要加入：

1. C 启动 Sync Copy → 点「加入」
2. C 填入 A 或 B 任一台的地址
3. **A 和 B 上同时弹审批框**
4. 任意一台上的用户点「同意」
5. A 和 B 的弹框都自动关闭，C 连上小组
6. C 自动与所有其它成员建立加密连接（不用再手动填 B 的地址）

**一次决定，全组生效。** 同样，如果任一台上点了「拒绝」，C 以后在这个会话内都无法加入（除非组内所有机器重启清空"封禁名单"）。

---

## 安全模型（简述）

- **没有密码**。身份由"审批弹框"背后的人类用户决定
- **端到端加密**：每对设备在握手时用 X25519 ECDH 协商出一把独立 AES-256 密钥。密文走 AES-GCM。同一 LAN 下即使有人抓包，只看得到密文
- **密钥只存内存**。应用退出，密钥丢失；下次启动重新协商（等价于每次会话一次性密钥）
- **同 WiFi 其它陌生设备**：握手阶段会走审批弹框，没人按同意就进不来
- **万一按错同意**：在任意已加入的机器上 `设置 → 退出应用`（或 `leave_group`）重新加入 → 内存里的 `approved_device_ids` 被清空

## FAQ

### Q: 为什么我点了「加入」一直转圈/超时？

- 对方没人在电脑前点「同意」→ 30 秒后超时
- 两台机器不在同一局域网 / WiFi（比如一台走手机热点一台走家里 WiFi）
- 对方机器的防火墙拦截了 5858 端口入站（在 Mac 系统设置→网络→防火墙 放行 Sync Copy；Windows 类似）
- 有 VPN 代理（如 Clash）劫持了 LAN 流量。临时关闭 VPN 或配置"局域网不代理"

### Q: 底部显示的 IP 是 `198.18.x.x` / `172.20.x.x` 之类奇怪地址

应用会优先选择 `192.168.*` > `10.*` > `172.16-31.*` 这些真实 LAN 地址。如果仍然显示奇怪地址，可能是：
- Clash / ClashX 的 fake-IP 模式（退出 VPN 重启即可）
- Docker / WSL / VMware 的虚拟网卡（已内建过滤，但个别命名未覆盖到）

### Q: 剪切板没有同步？

1. 两边的连接状态都是 `小组 N 台`（绿色）吗？不是的话先重新加入
2. 文本超过 1 MB / 图片超过 5 MB → 应用会跳过不传
3. 有些应用（比如某些密码管理器）会把内容标记为"敏感，不进剪切板"，这种拦不下来

### Q: 文件传过去没反应？

- 文件超过 5 MB → 发送端会直接红色横幅提示，不发送
- 对方 30 秒没按「保存」 → 自动视为拒绝
- 保存失败（磁盘满 / Downloads 目录被锁） → 接收端历史里会显示红色 `保存失败：<原因>`

### Q: 我在 A 上清空历史，B 上却还有？

- 请确认 A 和 B 都运行的是**同一版本**（或兼容版本）。较旧版本没有这个同步功能
- 如果仍有问题，打开 A 上运行终端的日志（`tracing::info!` 会输出 `clear-history broadcast ok peer=...`），看广播是否成功

### Q: 能跨网段 / 跨 VLAN 吗？

不能。工具是为"同网段 LAN"设计的。跨 VLAN / 跨路由器 / 跨互联网需要 VPN 或 NAT 打洞，本工具不做。

### Q: 有 CLI 版本吗？命令行？

没有。本工具设计为"浮窗置顶 + 剪切板自动同步"的桌面工具，不是开发者 CLI。

### Q: 会后台运行吗？耗电吗？

- 永远在前台显示浮窗（透明窗口）
- 闲置时 CPU 占用 < 1%，每秒一次剪切板读+哈希，每 10 秒一次心跳 ping
- 内存占用约 50-100 MB（一个 WebView 窗口的基础开销）

### Q: 数据留在哪里？会不会上传云端？

- **不会**。没有任何外部服务调用，所有流量都是 `http://192.168.x.y:5858`
- 本地配置：`~/Library/Application Support/com.synccopy.app/config.json`（Mac）或 `%APPDATA%\com.synccopy.app\config.json`（Windows）—— 只包含设备名、端口、device_id 这种不敏感字段
- 剪切板历史：仅在进程运行时内存里，退出即丢
- 加密密钥：只在内存里，永不落盘

## 键盘快捷键（系统级 Cmd+C / Ctrl+V）

无应用自己的快捷键。剪切板同步自动发生，你只用正常用系统的复制/粘贴快捷键即可。

## 系统要求

- macOS 11 Big Sur 或更新
- Windows 10 版本 1809 或更新（需 WebView2 Runtime，Win11 自带）
- 两台设备需要在同一局域网

---

## 已知限制

1. 单文件 5 MB 上限（之后可能放开到分片传输，暂无计划）
2. 应用重启后所有信任/封禁记录清空，需重新审批
3. 不支持跨 LAN；未适配 iOS/Android
4. 首次从某个不认识的设备收到握手时，必须有人在线审批——不适合"纯后台无人值守"的场景
